# Marvis Research — 深度研究框架

## 适用场景

- 深度调研（如"AI 芯片产业链现状"）
- 对比分析（如"A vs B 技术方案对比"）
- 行业综述（如"2026 年新能源车市场格局"）
- 论文检索（如"Transformer 架构最新改进"）

不适用于：天气/汇率/股价等简单事实查询。

## 研究流程

```
确定问题 → 多角度搜索 → 提取关键页面 → 交叉验证 → 结构化总结
```

## 各阶段操作

### 1. 多角度搜索
同一问题用 2-3 组不同关键词搜索，覆盖不同来源和视角：
- 中文关键词 + 英文关键词各搜一轮
- 搭配"对比"、"最新"、"趋势"、"问题"等修饰词扩面

### 2. 提取关键页面
对搜索结果中的高价值链接用 `web_fetch` 提取全文内容（Markdown 格式）。

### 3. 交叉验证
- 关键数据点至少两个独立来源确认
- 不一致的数据标注"存在分歧"
- 单一来源的信息标注"待交叉验证"

### 4. 结构化总结

输出格式：

```
## 核心结论
一句话总结。

## 关键发现
- 发现 1（来源）
- 发现 2（来源）

## 详细分析
| 维度 | 信息 | 来源 |
|------|------|------|
| ... | ... | ... |

## 局限说明
- 本次研究基于当前可获取的公开信息
- 不确定的信息已标注"待验证"

## 来源
1. [标题](URL)
2. [标题](URL)
```

## 效率原则

- 搜索结果不相关时果断重搜，不在无效页面上浪费时间
- 同类页面内容雷同时只提取 1-2 个代表性页面
- 信息量足够即停止搜索，不做过度采集
